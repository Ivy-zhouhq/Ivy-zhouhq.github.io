Defining functions

When a function is defined, variables are formal parameters. When calling a function, the actual parameters are passed.

When writing a function, if you find that it performs too many tasks, try to divide the code into two functions. Don’t forget, you can always call one function from another function, which helps to divide a complex task into a series of steps.

Store functions in modules

Store the function in a separate file called a module, and then import the module into the main program. The import statement allows the code in the module to be used in the currently running program file.

Import the entire module
To make the function importable, you must first create a module.
A module is a file with a .py extension and contains the code to be imported into the program.

pizza.py

Create another file named making_pizzas.py in the directory where **pizza.py is located. This file imports the newly created module, and then call make_pizza() twice

Function Writing Guide

When specifying default values ​​for formal parameters, do not have spaces on either side of the equal sign. For keyword arguments in function calls, this convention should also be followed
