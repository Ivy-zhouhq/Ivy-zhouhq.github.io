1 Create and use classes

The functions in the class are called methods
init() is a special method, Python will automatically run it whenever you create a new instance based on the Dog class.

When Python calls this init() method to create a Dog instance, it will automatically pass in the actual parameter self. Each method call associated with the class automatically passes the actual parameter self, which is a reference to the instance itself, allowing the instance to access the properties and methods in the class. When we create a Dog instance, Python will call the method init() of the Dog class. We will pass the name and 17 age to Dog() through the actual parameters; self will pass it automatically, so we don’t need to pass it. Whenever we create an instance based on the Dog class, we only need to provide values ​​for the last two formal parameters (name and age).
Variables that can be accessed through an instance are called attributes.

2 Inheritance

When creating a subclass, the parent class must be included in the current file and be in front of the subclass.

If a property or method is available in any car and not unique to electric cars, it should be added to the Car class instead of the ElectricCar class. In this way, people who use the Car class will get the corresponding functions, while the ElectricCar class only contains code that handles the specific attributes and behavior of electric vehicles.

Rewrite the parent class
As for the method of the parent class, as long as it does not conform to the behavior of the object simulated by the subclass, it can be rewritten. To this end, you can define a method in the subclass that has the same name as the superclass method to be overridden. In this way, Python will not consider the method of the parent class, but only focus on the corresponding method you defined in the subclass.

5 Coding style

The class name should adopt the camel case nomenclature, that is, the first letter of each word in the class name should be capitalized without underscore. Both the instance name and the module name are in lower case, with underscores between words.
For each class, a docstring should be included immediately after the class definition.
